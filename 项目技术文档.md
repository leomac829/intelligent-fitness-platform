# 智能健身管理平台 — 项目技术文档

> 本文档仅记录项目**已实现且实际调用**的功能，冗余代码、未注册路由、未调用函数均已排除。

---

## 一、项目概况

| 项目 | 说明 |
|------|------|
| 项目名称 | 智能健身管理平台（Fitness App） |
| 架构模式 | 前后端分离 B/S 架构 |
| 前端技术 | React 18.2 + TypeScript 5.2 + Ant Design 5.11 + ECharts 5.4 + Redux Toolkit 1.9 + Tailwind CSS 3.3 + Vite 5.0 |
| 后端技术 | Node.js + Express 4.18 + TypeScript 5.3 + Sequelize 6.35 + PostgreSQL |
| 认证方式 | JWT（jsonwebtoken 9.0），有效期24小时，bcrypt加密密码（salt rounds=10） |
| 前端端口 | 8888（Vite dev server） |
| 后端端口 | 3002（Express） |
| API规范 | RESTful API，JSON数据格式 |

## 二、系统架构

```
┌──────────────────────────────────────────────────────────┐
│                     前端应用 (SPA)                        │
│  React 18 + TypeScript + Ant Design + ECharts + Redux    │
│  构建工具: Vite 5  |  路由: React Router v6               │
│  端口: 8888  |  代理: /api/* -> localhost:3002            │
└────────────────────────┬─────────────────────────────────┘
                         │ HTTP (JSON)
┌────────────────────────▼─────────────────────────────────┐
│                  后端 API 服务 (Express)                   │
│  中间件链: CORS → Helmet → Morgan → JSON解析 → JWT认证    │
│  ORM: Sequelize 6  |  端口: 3002                          │
│  文件上传: multer  |  视频处理: ffmpeg                     │
└────────────────────────┬─────────────────────────────────┘
                         │ Sequelize ORM
┌────────────────────────▼─────────────────────────────────┐
│                  PostgreSQL 数据库                         │
│  19张实际使用的表  |  UUID主键  |  JSONB/ARRAY高级类型     │

└──────────────────────────────────────────────────────────┘
```

## 四、前端各页面详细功能

### 4.1 Login（登录页）

**已实现功能：**
- 用户名+密码登录（Redux `loginUser` thunk → `POST /api/auth/login`）
- 登录成功后 JWT 存入 localStorage，跳转首页
- 玻璃拟态（Glassmorphism）UI设计

**未实现（仅UI占位）：**
- Google/Apple/Meta 社交登录按钮 → `console.log` 仅打印
- "忘记密码？"链接 → `href="#"` 无功能

### 4.2 Register（注册页）

**已实现功能：**
- 注册表单：username、email、password、name、gender(男/女/其他)、age、fitness_level(初级/中级/高级)、height、weight
- 注册成功后自动登录并跳转首页（Redux `registerUser` thunk → `POST /api/auth/register`）

**未实现（仅UI占位）：**
- Google/Apple/Meta 社交注册按钮 → `console.log` 仅打印

### 4.3 Dashboard（主仪表盘）

**已实现功能（共12个子模块）：**

#### (1) 首页概览
- 统计卡片：总训练次数（含环比）、消耗热量（含环比）、连续天数
- 训练日历：自定义日历组件，支持年月切换，日期上标注训练部位标签
- 日程管理：创建/编辑/删除日程，选择训练部位（胸/背/肩/手臂/腹肌/臀/腿）
- 健身目标：体重目标进度条、月度训练目标、月度热量消耗、训练类型分布

#### (2) 数据分析
- 身体数据趋势图（ECharts）：体重、体脂率、肌肉量、BMI、腰围折线图
- 支持选择不同指标切换显示

#### (3) 训练记录
- 训练记录列表展示（日期、时长、热量、动作详情）
- 创建训练记录（Modal表单：日期、时长、备注、训练动作列表）
- 编辑/删除训练记录

#### (4) 视频库
- 嵌入 TrainingExercises 组件，展示训练动作库

#### (5) 训练日记
- 日记列表展示
- 创建日记（日期、内容、时长、标签）
- 编辑/删除日记

#### (6) 减脂计划
- 嵌入 FatLossPlan 组件

#### (7) 身体数据
- 身体数据列表
- 添加身体数据（体重、体脂率、肌肉量、胸围、腰围、臀围、二头肌围、大腿围）
- 编辑/删除身体数据
- 趋势图展示

#### (8) 个人资料
- 编辑基本信息（name、gender、age、height、weight、fitness_level）
- 头像上传

#### (9) AI健身助手
- 聊天界面，区分 user/assistant 消息
- 快捷问题："如何制定减脂计划？"、"初学者如何开始？"、"健身饮食建议"
- 流式响应展示（含进度条和思考状态）

#### (10) 目标管理
- 嵌入 GoalManagement 组件

#### (11) 日程安排
- 日历视图，按月查看日程
- 创建/删除日程

#### (12) 训练计划
- 嵌入 TanChengyiThreeSplit 组件

### 4.4 VideoLibrary（视频库）

**已实现功能：**
- 视频库列表浏览（动作文件夹 + 封面网格 + 视频数量统计）
- 搜索过滤（按名称和描述）
- 文件夹封面 hover 可直接播放预览
- 视频播放弹窗（Modal），支持视频/图片类型判断

**API调用：**
- `GET http://localhost:3002/api/video-library` — 加载动作列表
- `GET http://localhost:3002/api/video-library/{id}/videos` — 加载视频列表

### 4.5 ExerciseDetail（动作详情）

**已实现功能：**
- 展示动作图片/GIF、主要肌群、次要肌群、器械、分类、目标、动作说明

**API调用：**
- `GET http://localhost:3002/api/training-exercises/data` — 加载全部动作后前端按id筛选

### 4.6 TrainingLogs（训练日志）

**已实现功能：**
- 日志列表展示（日期、时长、训练项数量、笔记）
- 创建日志（Modal表单：date、duration、notes、训练动作列表）

**未实现（仅UI占位）：**
- 编辑按钮 → 无 onClick
- 删除按钮 → 无 onClick

### 4.7 LogDetail（日志详情）

**已实现功能：**
- 展示训练日期、时长、笔记、按动作分组的训练项表格（组数/次数/重量）

**未实现（仅UI占位）：**
- 导出按钮 → 无 onClick
- 编辑按钮 → 无 onClick
- 删除按钮 → 无 onClick

### 4.8 Statistics（训练统计）

**已实现功能：**
- 时间范围切换（week/month/year）
- 统计卡片：训练次数、训练总时长、总训练量、平均强度
- 训练频率柱状图（ECharts bar）
- 训练量趋势折线图（ECharts line, smooth）
- 肌肉群训练分布饼图（ECharts pie）

**数据问题：**
- 肌肉群分布数据为硬编码模拟数据，未使用API返回的真实数据
- 训练频率数据为全零数组，未填充真实数据
- 训练量趋势为简单等分计算，非真实趋势

### 4.9 Profile（个人资料）

**已实现功能：**
- 双标签页："基本资料" 和 "修改密码"
- 基本资料编辑：name、gender、age、height、weight、fitness_level
- 密码修改：oldPassword + newPassword（最少6位）

### 4.10 FatLossPlan（减脂饮食计划）

**已实现功能（完整实现）：**
- 输入信息：性别、体重、身高、年龄、训练时长、训练强度、当前体重、目标体重、营养素比例(532/442)
- 计算结果：BMR(基础代谢)、TDEE(总消耗)、每日建议摄入热量、三大营养素(碳水/蛋白/脂肪)、每周减重量、总周期
- 多套食谱方案切换
- "换一批"重新生成食谱
- 点击食物项可替换同类食物
- 加载历史计划

**API调用：**
- `POST http://localhost:3002/api/fatloss/calculate` — 计算减脂计划
- `GET http://localhost:3002/api/fatloss/latest` — 获取最近计划
- `POST http://localhost:3002/api/fatloss/replace-food` — 替换食物

### 4.11 TrainingExercises（训练动作库）

**已实现功能：**
- 分页浏览（PAGE_SIZE=48，Ant Design Pagination）
- 搜索过滤（按名称/器材/肌肉群）
- 按器材筛选
- 详情弹窗（Modal：GIF/图片、说明、肌群、器械、分类）
- 响应式网格（2-8列动态计算）
- localStorage缓存（30分钟过期）

**API调用：**
- `GET http://localhost:3002/api/training-exercises/data` — 加载训练动作

### 4.12 TanChengyiThreeSplit（三分化训练计划）

**已实现功能：**
- 配置表单：健身基础(新手/中级/高级)、训练目标(增肌/力量/减脂)、每周训练天数(3/6)、基础重量(kg)、渐进超负荷率(5-10%)、计划周期(1-52周)
- 生成推/拉/腿三分化训练计划
- 周次切换（多周计划前后切换）
- 自动加载最近保存的计划

**API调用：**
- `GET http://localhost:3002/api/three-split-plans/latest` — 加载历史计划
- `POST http://localhost:3002/api/three-split-plans/generate` — 生成计划

### 4.13 AdminDashboard + UserManagement + VideoManagement（管理后台）

**已实现功能：**

用户管理：
- 用户列表（表格：ID、用户名、邮箱、姓名、性别、年龄、健身水平、创建时间）
- 添加用户（`POST http://localhost:3002/api/auth/register`）
- 编辑用户（`PUT http://localhost:3002/api/auth/users/{id}`）
- 删除用户（`DELETE http://localhost:3002/api/auth/users/{id}`）

视频库管理：
- 视频库列表（表格：名称、视频数、创建时间）
- 添加/编辑/删除视频库条目
- 视频上传（Ant Design Upload.Dragger，最多10个，500MB限制）
- 视频文件查看和删除

## 五、前端状态管理

### 5.1 Redux Store 结构

```typescript
{
  user: {
    user: any,           // 当前登录用户信息
    token: string | null, // JWT令牌（初始从localStorage读取）
    loading: boolean,
    error: string | null
  },
  log: {
    logs: any[],         // 训练日志列表
    log: any | null,     // 当前日志详情
    stats: any | null,   // 训练统计数据
    loading: boolean,
    error: string | null
  }
}
```

### 5.2 userSlice — 实际使用的异步Thunk

| Thunk | API端点 | 调用位置 |
|-------|--------|---------|
| `registerUser` | `POST /api/auth/register` | Register.tsx |
| `loginUser` | `POST /api/auth/login` | Login.tsx |
| `getUserProfile` | `GET /api/auth/profile` | App.tsx, Dashboard.tsx, FatLossPlan.tsx |
| `updateProfile` | `PUT /api/auth/profile` | Profile.tsx, FatLossPlan.tsx |
| `changePassword` | `PUT /api/auth/password` | Profile.tsx |
| `logoutUser` | 无API（清localStorage） | Dashboard.tsx |

### 5.3 logSlice — 实际使用的异步Thunk

| Thunk | API端点 | 调用位置 |
|-------|--------|---------|
| `getLogs` | `GET /api/logs` | TrainingLogs.tsx |
| `getLogDetail` | `GET /api/logs/{id}` | LogDetail.tsx |
| `createLog` | `POST /api/logs` | TrainingLogs.tsx |
| `getStats` | `GET /api/logs/stats` | Statistics.tsx |

### 5.4 自定义Hook

#### useDataCache
- **使用位置**：Dashboard.tsx
- **功能**：基于内存的数据缓存，默认TTL 5分钟
- **实际使用的缓存键**：`DASHBOARD`、`WORKOUTS`
- **实际调用的方法**：`getCached`、`setCache`、`invalidate`

#### useGoalManagement
- **使用位置**：Dashboard.tsx
- **功能**：目标管理业务逻辑封装
- **API调用**：`GET /api/goals`、`POST /api/goals`、`PUT /api/goals/{id}`、`DELETE /api/goals/{id}`
- **联动**：保存/删除目标后自动刷新仪表盘数据

## 八、核心业务逻辑详解

### 8.1 减脂饮食计划计算引擎

#### 计算流程

```
用户输入 → BMR计算 → TDEE计算 → 每日热量计算 → 宏量营养素分配 → 减脂周期计算 → 食谱生成
```

#### BMR计算（Mifflin-St Jeor公式）

```
男性: BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5
女性: BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161
```

#### TDEE计算

```
TDEE = BMR + 训练时长(分钟) × 强度系数
强度系数: beginner=5, intermediate=8, advanced=10 (kcal/分钟)
```

#### 每日热量计算

```
每日热量 = max(TDEE - 400, BMR)  // 热量缺口400kcal，但不低于BMR
```

#### 宏量营养素分配

| 比例类型 | 碳水化合物 | 蛋白质 | 脂肪 |
|---------|-----------|--------|------|
| 532 | 50% | 30% | 20% |
| 442 | 40% | 40% | 20% |

换算：碳水/蛋白 4kcal/g，脂肪 9kcal/g

#### 减脂周期计算

```
月减重率 = 当前体重 × 4%
总月数 = (当前体重 - 目标体重) / 月减重率
总周数 = 总月数 × 4.33
```

#### 食谱生成算法

1. 从数据库按6类（主食/蛋白质/蔬菜/乳制品/蛋类/油脂类）随机获取食材
2. 三餐热量分配：早餐30%、午餐40%、晚餐30%
3. 每餐构建流程：
   - 按餐型分配食物种类和份量
   - 碳水偏差>20%时调整主食份量
   - 蛋白质偏差>20%时调整蛋白质份量
   - 脂肪不足时补充油脂类食物
   - 蔬菜填补剩余热量
4. 食物替换：从同分类中随机选不同食物，保持份量不变

### 8.2 三分化训练计划生成

#### 配置参数

| 参数 | 可选值 | 默认值 |
|------|--------|--------|
| 健身基础 | beginner/intermediate/advanced | intermediate |
| 训练目标 | muscle_gain/strength/fat_loss | muscle_gain |
| 每周训练天数 | 3/6 | 3 |
| 基础重量(kg) | 数值 | 50 |
| 渐进超负荷率 | 5%-10% | 5% |
| 计划周期 | 1-52周 | 4 |

#### 训练日分配

| 训练日类型 | 训练肌群 | 颜色标识 |
|-----------|---------|---------|
| Push（推日） | 胸肌、肩部、三头肌 | 蓝色 |
| Pull（拉日） | 背部、二头肌 | 紫色 |
| Legs（腿日） | 腿部、臀部 | 绿色 |

#### 渐进超负荷

每周训练重量按配置的超负荷率递增：
```
第N周重量 = 基础重量 × (1 + 超负荷率/100)^(N-1)
```

#### 数据存储

使用 PostgreSQL JSONB 类型存储 config 和 plan_data，每用户仅保留一个最新计划。

### 8.3 身体数据智能继承

创建新记录时，未填写的字段自动从该用户最近一条记录继承：

```typescript
const latestMeasurement = await BodyMeasurement.findOne({
  where: { userId },
  order: [['date', 'DESC']]
});
if (latestMeasurement) {
  ['bodyFat', 'muscleMass', 'chest', 'waist', 'hips', 'biceps', 'thighs'].forEach(field => {
    if (!measurement[field]) {
      measurement[field] = latestMeasurement[field];
    }
  });
}
```

### 8.4 目标自动完成判断

| 目标类型 | 完成条件 |
|---------|---------|
| 体重目标（减脂方向） | current_value <= target_value |
| 体重目标（增肌方向） | current_value >= target_value |
| 训练目标 | current_value >= target_value |
| 热量目标 | current_value >= target_value |

### 8.5 视频封面自动生成

上传视频时，使用 ffmpeg 自动截取视频第1秒画面作为封面：
- 分辨率：320×180
- 支持格式：MP4, MOV, AVI, MKV
- 大小限制：单个500MB，每次最多10个

## 九、认证与安全机制

### 9.1 JWT认证流程

```
注册/登录 → 服务器生成JWT({ userId }, JWT_SECRET, { expiresIn: '24h' })
         → 返回token给前端
         → 前端存入localStorage
         → 后续请求携带 Authorization: Bearer <token>
         → authenticateToken中间件验证
         → jwt.verify解码获取userId
         → User.findByPk查询用户
         → req.user = user（挂载到请求对象）
```

### 9.2 密码安全

- 加密算法：bcrypt
- Salt rounds：10
- User模型提供 `setPassword(password)` 和 `validatePassword(password)` 方法

### 9.3 安全中间件

| 中间件 | 功能 |
|--------|------|
| cors() | 处理跨域请求 |
| helmet() | 设置安全HTTP头（含CSP指令） |
| morgan('combined') | 记录请求日志 |

## 十三、已知问题与限制

### 13.1 路由顺序BUG

以下API端点因定义在参数路由 `/:id` 之后，可能被错误拦截：

| 文件 | 问题端点 | 影响 |
|------|---------|------|
| logs.ts | GET /stats | 统计数据无法获取 |
| workout.ts | GET /stats/summary | 训练统计汇总无法获取 |
| bodyMeasurement.ts | GET /stats/summary | 身体数据统计无法获取 |
| foodNutrition.ts | GET /search/exact | 精确搜索无法使用 |

### 13.2 安全问题

| 问题 | 影响 |
|------|------|
| 视频库写操作无认证 | 任何人可创建/删除视频库内容 |
| 管理员API无权限校验 | 任何登录用户可管理其他用户 |
| 部分API直接写死localhost URL | 生产环境部署会失效 |

### 13.3 数据问题

| 问题 | 影响 |
|------|------|
| Statistics页面图表数据为假数据 | 统计图表展示不准确 |
| TrainingLogs编辑/删除按钮无功能 | 用户无法编辑或删除日志 |
| LogDetail导出/编辑/删除按钮无功能 | 用户无法操作日志详情 |
| 两套训练记录系统(Workout/TrainingLog)数据不互通 | 训练数据分散 |
